# 전농15 · 정비사업 현황 대시보드

전농15구역(서울 동대문구 전농동 152-65 일대) 재개발(신속통합기획)의 **진행현황·소유자 할 일·고시공고·구역 소식지**를 한 화면에서 제공하는 단일 페이지 대시보드입니다. 별도 서버·빌드 없이 `index.html` 하나로 동작합니다.

> ⚠️ 본 페이지는 데모 버전입니다. 기본정보·진행현황 수치 일부는 샘플이며, 고시/공고는 정비몽땅·서울정보소통광장의 검증된 실제 원문으로 연결됩니다. 투자·법적 판단의 근거로 사용하지 마세요.

## 🔗 라이브 데모
GitHub Pages 배포 후 아래 형식의 주소로 접속합니다.

```
https://<GitHub-사용자명>.github.io/<저장소명>/
```

## ✨ 주요 기능 (현재 버전)
- **진행현황 타임라인** — 대상지선정 → 정비구역지정 → 추진위원회 승인 → 조합설립인가 → 사업시행인가 → 관리처분인가 → 착공. 정비구역지정·추진위원회 승인은 동시 투트랙으로 "준비 중" 표시.
- **단계 클릭 아코디언** — 각 단계를 누르면 해당 단계의 세부 일정이 펼침/접힘. 단계별로 자기 일정만 보유.
- **소유자 TO-DO Checklist** — 진행 중 단계에서 토지등소유자가 챙겨야 할 일을 상단에 강조. 마감일(D-day) 기준 표시이며, 전 항목 마감 경과 시 자동 비표시.
- **자동 상태/D-day 산정** — 일정의 시작·종료일과 오늘 날짜를 비교해 완료/진행/예정과 D-day(마감일 기준)를 자동 계산.
- **고시·공고·결재** — 최신 공지 1건 상단 고정 + 이하 최신순. 정비몽땅·서울정보소통광장의 검증된 원문으로 직접 이동.
- **구역 소식지** — 이미지 업로드(설명 500자), 앨범 미리보기(대표+썸네일), 사진 클릭 시 설명·확대.
- **확장 구조** — 데이터 레이어(`REGIONS` + `CURRENT_ID`)만 추가하면 다구역으로 확장 가능.

## 🚀 GitHub Pages 배포 방법
1. 새 public 저장소를 만들고 `index.html`(필수)과 `README.md`, 문서 파일을 업로드(또는 push).
2. 저장소 **Settings → Pages** 이동.
3. **Build and deployment → Source**를 `Deploy from a branch`로, **Branch**를 `main` / `/ (root)`으로 설정 후 저장.
4. 1~2분 뒤 안내되는 `https://<사용자명>.github.io/<저장소명>/` 주소로 접속하면 대시보드가 공개됩니다.

CLI 예시:
```bash
git init
git add index.html README.md *.md
git commit -m "전농15 정비사업 현황 대시보드 공개"
git branch -M main
git remote add origin https://github.com/<사용자명>/<저장소명>.git
git push -u origin main
```
